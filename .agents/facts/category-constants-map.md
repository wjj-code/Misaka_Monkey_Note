# 分类常量映射

## 操作流程（v1.0）

```
合并页: 左上角支出/收入切换 + 类别网格（默认支出）
   ↓
Level 2: 子项目列表
   ↓
Level 3: 记账表单
```

> ⚠️ **一级分类常量已内嵌 emoji 前缀**（如 `"🍚 吃"`），下表「常量」列即字符串实际值；`category1` 入库也是该值。另有 `public static Map<String,String> CAT_EMOJI` 把带文字分类映射为纯 emoji，供 `BillAdapter` 取图标。

## 支出分类 (CAT1_EXPENSE)

> 定义位置：`CategoryCatalog`（与 `AddTransactionSheet` 内嵌常量一致）、`SUBS_EXPENSE`

| 常量（字符串实际值） | 子项目列表 |
|------|-----------|
| `🍚 吃` | 早餐、午餐、晚餐、零食、水果、饮品、聚餐 |
| `🏠 住` | 房租、水电、物业、网费、日用品、维修 |
| `🎮 娱` | 电影、游戏、KTV、旅游、运动、订阅会员 |
| `🚗 交通` | 公交、地铁、打车、加油、停车、高铁/飞机 |
| `🛒 购物` | 服饰、数码、美妆、家居、超市 |
| `📚 教育` | 书籍、课程、培训、文具、考试费 |
| `🏥 医疗` | 门诊、药品、体检、牙科 |
| `📌 其他` | 自定义（弹出输入框自由输入） |

## 收入分类 (CAT1_INCOME)

> 定义位置：`AddTransactionSheet.CAT1_INCOME[]`、`SUBS_INCOME`

| 常量（字符串实际值） | 子项目列表 |
|------|-----------|
| `💼 工资` | 基本工资、奖金、加班费、补贴 |
| `📈 投资` | 股票、基金、利息、分红、房租收入 |
| `💻 兼职` | 副业、稿费、咨询费、设计费 |
| `↩️ 退款` | 购物退款、报销、押金退还 |
| `🎁 礼金` | 红包、礼金、赠与 |
| `📌 其他` | 自定义（弹出输入框自由输入） |

## 财务分类 (category3)

> 定义位置：`CAT3_LABELS`（在 BillAdapter 和 DailyDetailAdapter 中各有定义）  
> ⚠️ **仅支出 (type=0) 使用并必选**；切换到收入时该区隐藏，`category3` 置 `-1`，列表/明细也不显示分类标签。

| 常量值 | 标签 | 颜色 | 说明 |
|--------|------|------|------|
| 0 | 维持类 | `#2196F3` (蓝) | 生活必需品 |
| 1 | 消费类 | `#FF9800` (橙) | 非必需消费 |
| 2 | 提升类 | `#9C27B0` (紫) | 自我提升 |
| 3 | 社交类 | `#00BCD4` (青) | 人际交往 |

## 收支类型 (type)

| 常量值 | 含义 | 颜色 |
|--------|------|------|
| 0 | 支出 | `#E53935` (红) |
| 1 | 收入 | `#4CAF50` (绿) |

## 添加新分类步骤

1. 在 `CAT1_EXPENSE` 或 `CAT1_INCOME` 数组添加一级分类名（**带 emoji 前缀**，如 `"🎵 音乐"`）
2. 在对应的 `SUBS_EXPENSE` 或 `SUBS_INCOME` Map 中添加子项目列表（key 用带 emoji 的同一字符串）
3. 在 `CAT_EMOJI` 映射表补一条「带文字 → 纯 emoji」，否则账单列表图标会回退为默认 `💰`
4. `buildCategoryGrid()` 自动读取数组动态生成按钮，自动计算 `rowCount`
5. **注意**：GridLayout 4 列布局，收入 6 类 = 2 行，支出 8 类 = 2 行
