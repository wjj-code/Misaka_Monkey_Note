# 呱太记账 — 产品与技术说明

> 版本 v1.0 | 最后更新 2025-06-11

## 产品概述

「呱太记账」是一款面向个人用户的轻量级记账工具，目标用户是希望快速记录日常收支、对消费有基本追踪需求的人群。核心设计理念：**四步完成记账**——选收支类型 → 选类别 → 选子项目 → 填金额确认，零学习成本（前两步合并在同一页，实际为三级递进面板）。

### 🎨 设计风格
- **配色**：柔和蓝色主色调 (#2D9CDB)，温暖浅灰背景 (#F2F4F7)，白色卡片
- **卡片化**：MaterialCardView 全局圆角卡片（12~16dp），带细微描边和阴影
- **图标**：分类带 emoji 图标（🍚吃 🏠住 🎮娱 📚教育 🚗交通 🛒购物 🏥医疗），emoji 已内嵌进分类常量字符串（如 `"🍚 吃"`）
- **字体**：标题 26sp Bold，正文 15sp，金额 18~22sp Bold，层次分明

## 已实现功能

### 底部导航
5 个一级入口，覆盖记账场景的全部操作闭环：

| 栏目 | 功能 |
|------|------|
| **账户** | 活期/资产/全部筛选（默认活期）+ 增删改/设默认；新增后自动切到对应类型 |
| **账单** | **顶部预算+收支汇总栏**（随列表滚动切换对应月份）+ **全部账单**（按月分段 + 按日分组），左滑删除、点击编辑 |
| **＋** | 点击弹出记账面板（Level3 可选入账账户）；**长按**进入语音记账 |
| **日历** | 月历视图，每日标注支出金额，点击查看当天明细 |
| **我的** | 用户卡片 + 功能列表：**预算设置已实现**（按月设月预算，SharedPreferences 存储），数据导出/关于为预留入口 |

### 记账流程（三级递进面板）
用户点击底部「＋」后，从屏幕底部弹出固定高度面板（约 **42% 屏幕**，长屏下过窄会挤），三级递进式引导：

1. **收支类型 + 类别（合并页）**：面板上方左侧紧凑排布「支出」「收入」切换按钮（默认选中支出），下方展示对应类目网格（支出8类/收入6类）
2. **选择子项目**：与 Level 1 同款 4 列小模块网格；每类 4~7 个细分项，一页放不下可滚动，「其他」支持自定义输入
3. **填写详情**：
   - 0-9 数字键盘（固定在面板底部，方便单手输入），支持小数点
   - 财务四分类（维持类 / 消费类 / 提升类 / 社交类）**仅支出必选**，切到收入时该区隐藏且不校验
   - 日期时间默认显示当前时刻（**不展开选择器，节约空间**），点击弹出**三列滚轮选择器**（月日周 | 时 | 分，老虎机风格）
   - 备注选填

### 语音记账（长按 ＋）
- 在输入框内用**系统键盘自带语音转文字**（或手动输入），点「解析预览」查看字段拆分
- 可说/输入：收支、类别、细则、金额、备注、账户（照读页内示例句）
- 必填未齐 → 跳转「补全账单」；必填已齐但选填未填 → 保存后浮动提示
- 备研：App 内麦克风方案见 `VoiceInputMicLegacy`（已停用）

### 收支图表（账单页入口）
- 饼图：默认**当月**，可切换近半年/近1年；支出按财务四类→一级类→二级子项三级折叠展示占比，收入按一级类→二级子项两级
- 折线图：默认**近 6 个月**月度趋势，可切换当月/半年/1年；与支出/收入切换联动，始终以当前月为时间终点

### 数据管理
- 账单展示**全部月份**，按月分段 + 按日分组；滚动时顶部汇总跟随当前可见月份
- **点击任意记录进入编辑页**（全字段问卷式表单），修改后保存
- **左滑删除**：左滑后弹出确认对话框，点击「删除」才实际删除，防止误操作
- 日历：每日格子标注支出；**当月**自动选中今天，**其他月**自动选中该月最早有账日；点击顶部年月可快捷跳转
- 点击下方列表展示当天明细（与账单同款卡片，仅展示）
- 记账成功、编辑或删除后，账单列表、日历、账户汇总全部自动同步刷新

### 预算管理
- **账单页顶部**与「我的」页均可设置月预算（点击「剩余 ¥」或「💰 预算设置」）
- 预算独立用 **SharedPreferences**（`budget_prefs`）按月（`budget_yyyy-MM`）持久化，**不进入 Room**
- 账单页实时计算「剩余 = 预算 − 当月支出」并显示
- `BudgetViewModel` 持有 `BudgetStorage`，结合 `MonthSummary.totalExpense` 计算剩余预算

### 账户/账本管理
- 「账户」页管理多账户：**添加 / 编辑 / 删除 / 设为默认**（长按弹菜单）
- 账户字段：名称、余额、类型（活期/资产）、卡号、备注 + 6 个开关（计入净资产 / 可支出 / 可收入 / 可转入 / 可转出 / 启用）
- 账户存于 **Room `accounts` 表**（DB v2 新增）；`Transaction.accountId` 关联账户，新记录默认 `0`（默认账户）

## 技术方案

### 架构：MVVM + Repository

```
 UI (Fragment / Activity)
        │  LiveData 观察
        ▼
 ViewModel (TransactionViewModel)
        │
        ▼
 Repository (TransactionRepository)  ← 数据抽象层，当前封装 Room，后续可替换为云端 API
        │
        ▼
 Room DAO → SQLite (ledger.db)
```

- **UI 层**：4 个 Fragment + 1 个 BottomSheetDialogFragment，通过 `ViewModelProvider` 获取 ViewModel，LiveData 自动驱动 UI 刷新
- **ViewModel 层**：`AndroidViewModel` 持有 Application，是 UI 与 Repository 的唯一桥梁（交易 `TransactionViewModel`、账户 `AccountViewModel`、预算 `BudgetViewModel`）
- **Repository 层**：封装所有数据操作，**读**返回 `LiveData`（可观察），**写**在单线程池异步执行，避免阻塞主线程（`TransactionRepository` / `AccountRepository`）
- **数据层**：Room 持久化（**DB v2**，`ledger.db`），`Transaction` 8 字段（含 `accountId`）+ `Account` 表；预算走 SharedPreferences。DAO 提供按日/月/范围查询及汇总聚合

### 版本管理

版本号集中定义于 `gradle.properties`（`APP_VERSION_CODE` / `APP_VERSION_NAME`），`build.gradle.kts` 动态读取。修改日志记录于 `facts/CHANGELOG.md`。

### 数据与代码分离设计

当前使用 Room（本地 SQLite）存储，但 Repository 层提供了统一的数据访问接口。未来如需迁移到云端：

1. 新增 `CloudTransactionRepository` 实现同一套接口
2. 在 `TransactionViewModel` 中切换注入的 Repository 实现
3. **数据库层（DAO/Entity/Database）无需任何修改**

### 关键设计决策

- **收支类型前置**：收入与支出分类不同，第一步先选类型，后续展示对应类目
- **写操作异步化**：所有 insert/update/delete 在独立 IO 线程执行，保证 UI 不卡顿
- **LiveData 驱动刷新**：一笔记账提交后，账单列表、日历、账户汇总全部自动更新，无需手动通知
- **三级面板纯 View 切换**：记账弹窗不涉及 Fragment 事务，用 `View.setVisibility()` 控制层级，零开销。初始合并页直接将收支切换与类别网格放在一起，默认支出。
- **数字键盘固定底部**：内嵌 3×4 数字键盘 + 退格 + 小数点，不调用系统输入法，单手操作友好
- **滚轮时间选择**：NumberPicker 实现时分滚轮拨选（老虎机风格），精确到分钟
- **弹窗高度一致**：BottomSheet 固定约 42% 屏幕高度，禁止拖拽，各级界面高度一致
